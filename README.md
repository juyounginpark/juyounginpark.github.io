# 📌 이모지콘 (EmojiCon)

나만의 커스텀 이모지콘을 뽑는 웹 게임.
50개 이모지 중 20개가 제시되고, 당신은 5개를 순서대로 고릅니다. 서버는 선택 순서 가중치를 반영해 성향 벡터(EN/TJ)를 계산하고, 4축의 부호(E±/N±/T±/J±)로 16가지 타입 중 하나를 결정해 모노톤 이모지콘을 생성합니다. 결과 화면에는 이모지콘, 한 문장 요약, 당신이 고른 5개 이모지의 한줄 설명, 그리고 팔레트가 제공됩니다. (벡터 수치는 클라이언트에 노출하지 않습니다.)

------

| 기능                           | 설명                                                                 |
|--------------------------------|----------------------------------------------------------------------|
| 20개 풀에서 5개 이모지 선택    | 서버가 제공한 20개 이모지 중 사용자가 5개를 순서대로 고름            |
| 순서 가중치 + 부스트 계산      | 1~3픽은 보너스 가중치 부여, 선택 순서에 따른 가중합으로 성향 벡터 산출 |
| 16 타입 이모지콘 결정           | 벡터 부호(E/N/T/J) 조합으로 16가지 타입 중 하나 선택, 모노톤 규칙 적용 |
| 한 문장 요약                   | 상위 3픽 + 스타일(plan, extraversion) 기반으로 자연어 요약 생성       |
| 5줄 설명                        | 사용자가 고른 5개 이모지의 oneLine을 선택 순서대로 출력               |
| 팔레트 반환                     | EN 사분면 기반 색상 팔레트 4개 제공 (페이지 테마용)                  |
| 공유용 링크                     | share_id 발급으로 결과 조회 및 공유 가능                              |

------


| 구분        | 사용 기술                         | 비고                                   |
|-------------|----------------------------------|----------------------------------------|
| Backend     | Spring Boot (Java)              | API 서버, 로직 처리                     |
| Database    | PostgreSQL / MySQL              | 세션(run) 및 결과 저장                  |
| Frontend    | React / Next.js                 | UI, 사용자 인터랙션                     |
| Infra/DevOps| GitHub, Docker (선택)           | 협업 및 배포 자동화                     |
| API 문서화  | Swagger / Postman               | API 테스트 및 문서화                    |
| 버전 관리   | GitHub                          | 백엔드·프론트 분리 개발 후 통합          |
